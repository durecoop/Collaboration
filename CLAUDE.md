# 팀 업무 관리 웹 프로그램

## 팀원 (= 서비스 사용자)
- 지원: 프로젝트 리더
- 소영: 팀원
- 슬기: 팀원

## 현재 단계: 니즈 파악 → 기획 → 설계 → 개발

## 프로젝트 목표
지원, 소영, 슬기 3명이 팀 업무를 관리하는 웹 프로그램을 직접 만든다.

## 서비스 핵심 요구사항
- 로그인: 접속 시 3명 중 누구인지 식별 (사용자 구분 필수)
- 각자의 업무를 등록, 확인, 관리할 수 있어야 함
- 팀 전체 업무 현황을 한눈에 볼 수 있어야 함

## 진행 과정
1. 니즈 파악: 각자 `docs/요구사항/` 에 원하는 기능 작성
2. 기획: 니즈 종합 → 기능 목록 정리 → 화면 설계
3. 설계: 기술 스택 선정, DB 구조, API 설계
4. 개발: 실제 웹 프로그램 구현
5. 테스트: 3명이 직접 사용해보며 피드백

# Claude 사용 가이드 (처음 쓰는 팀원을 위한 안내)

## 기본 사용법
1. 이 프로젝트 폴더에서 터미널을 열고 `claude` 입력
2. 말로 시키면 됩니다 (예: "회의록 작성해줘", "기획서 수정해줘")
3. 작업이 끝나면 "커밋하고 푸시해줘" 라고 말하기
4. 작업 시작할 때 "pull 해줘" 라고 말하기

## 자주 쓰는 명령어 (Claude에게 이렇게 말하세요)
| 하고 싶은 것 | Claude에게 할 말 |
|---|---|
| 최신 코드 받기 | "pull 해줘" |
| 내 작업 저장 | "커밋하고 푸시해줘" |
| 팀원이 뭐 했는지 보기 | "최근 변경사항 보여줘" |
| 브랜치 만들기 | "feature/내이름-작업내용 브랜치 만들어줘" |
| 충돌 해결 | "충돌 해결해줘" |
| main에 합치기 | "main에 머지해줘" |
| 서버에 배포하기 | "서버에 반영해줘" |

## 작업 순서 (매번 이렇게 하세요)
```
1. "pull 해줘"              ← 시작할 때 최신 내용 받기
2. 작업하기                  ← Claude에게 시키거나 직접 작업
3. "커밋하고 푸시해줘"        ← 끝나면 저장하고 올리기
```

## 서버 배포 방법
- 배포 서버: https://durecoop-team.onrender.com/
- Render가 GitHub `master` 브랜치와 연동되어 있음
- **master에 push하면 자동으로 서버에 배포됩니다**
- 배포가 안 될 때 확인사항:
  1. 내 작업이 `master` 브랜치에 머지/푸시되었는지 확인
  2. feature 브랜치에만 push하면 배포 안 됨 → master에 머지 필요
  3. Claude에게 "서버에 반영해줘" 라고 말하면 됩니다

## 주의사항
- 작업 시작 전에 반드시 pull 먼저!
- 같은 파일을 동시에 수정하면 충돌 날 수 있음 → 서로 작업 범위 공유하기
- .env 파일은 절대 커밋하지 않기 (비밀번호 등 포함)

# 프로젝트 규칙

## 문서 작성 규칙
- 문서는 한글로 작성
- 파일명은 영문 소문자, 단어 구분은 하이픈(-) 사용
- 날짜 포함 시: YYYY-MM-DD 형식 (예: 2026-03-17)

## 브랜치 규칙
- main: 완성된 코드만 (직접 수정 금지)
- feature/이름-작업내용: 각자 작업용 브랜치
  - 예: feature/jiwon-login, feature/soyoung-dashboard, feature/seulgi-task

## 커밋 메시지 규칙
- 한글로 작성
- 예: "로그인 페이지 기획서 작성", "DB 스키마 초안 추가"

## 폴더 구조
```
Collaboration/
├── CLAUDE.md                 ← 이 파일 (팀 규칙)
├── docs/                     ← 기획 문서
│   ├── 요구사항/
│   │   ├── jiwon.md          ← 지원 니즈
│   │   ├── soyoung.md        ← 소영 니즈
│   │   └── seulgi.md         ← 슬기 니즈
│   └── 회의록/
├── src/                      ← 소스 코드 (개발 시작하면 사용)
└── README.md
```
