# 웹 배포 설계 기획서

## 현재 상태

```
현재: prototype/index.html (단일 HTML 파일)
- 데이터: localStorage (각자 브라우저에만 저장)
- 문제: 지원이 등록한 업무를 소영/슬기가 볼 수 없음
```

## 목표

```
목표: 3명이 같은 데이터를 보고 함께 쓰는 웹 서비스
- 지원이 업무를 등록하면 → 소영/슬기도 바로 보임
- 소영이 공지를 읽으면 → 읽음 상태가 다른 사람에게도 반영
- 어디서든 접속 가능 (회사, 집, 폰)
```

### 데이터베이스 테이블 설계

```sql
-- 업무
CREATE TABLE tasks (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  title       TEXT NOT NULL,
  priority    TEXT DEFAULT 'normal',    -- 'urgent' | 'normal'
  status      TEXT DEFAULT '시작전',     -- '시작전' | '진행중' | '완료'
  deadline    TEXT,                      -- '2026-03-20'
  assignees   TEXT,                      -- '지원,소영' (쉼표 구분)
  created_by  TEXT,
  created_at  TEXT DEFAULT (datetime('now','localtime')),
  updated_at  TEXT DEFAULT (datetime('now','localtime'))
);

-- 업무 메모
CREATE TABLE task_memos (
  id       INTEGER PRIMARY KEY AUTOINCREMENT,
  task_id  INTEGER REFERENCES tasks(id) ON DELETE CASCADE,
  date     TEXT,
  content  TEXT,
  author   TEXT
);

-- 회의록
CREATE TABLE meetings (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  title      TEXT NOT NULL,
  date       TEXT,
  attendees  TEXT,          -- '지원,소영,슬기'
  agenda     TEXT,
  content    TEXT,
  created_at TEXT DEFAULT (datetime('now','localtime'))
);

-- 회의록 체크사항
CREATE TABLE meeting_checks (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  meeting_id INTEGER REFERENCES meetings(id) ON DELETE CASCADE,
  text       TEXT,
  who        TEXT,
  done       INTEGER DEFAULT 0
);

-- 공지
CREATE TABLE notices (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  title      TEXT NOT NULL,
  type       TEXT,           -- '타임세일' | '증정이벤트' | '공지'
  date       TEXT,
  author     TEXT,
  content    TEXT,
  created_at TEXT DEFAULT (datetime('now','localtime'))
);

-- 공지 읽음 상태
CREATE TABLE notice_reads (
  notice_id  INTEGER REFERENCES notices(id) ON DELETE CASCADE,
  user_name  TEXT,
  read_at    TEXT DEFAULT (datetime('now','localtime')),
  PRIMARY KEY (notice_id, user_name)
);

-- 개인 메모
CREATE TABLE memos (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  title      TEXT NOT NULL,
  content    TEXT,
  owner      TEXT,
  created_at TEXT DEFAULT (datetime('now','localtime'))
);

-- 고정 반복 업무
CREATE TABLE fixed_tasks (
  id    INTEGER PRIMARY KEY AUTOINCREMENT,
  day   INTEGER,    -- 1=월 ~ 5=금
  title TEXT,
  owner TEXT
);

-- 캘린더 일정
CREATE TABLE calendar_events (
  id    INTEGER PRIMARY KEY AUTOINCREMENT,
  date  TEXT,
  title TEXT,
  cat   TEXT,       -- 'task' | 'meeting' | 'leave' | 'event'
  who   TEXT
);
```

### API 설계

```
기본 URL: http://localhost:3000/api

업무 (Tasks)
  GET    /api/tasks              전체 업무 목록
  GET    /api/tasks/:id          업무 상세 (메모 포함)
  POST   /api/tasks              업무 등록
  PUT    /api/tasks/:id          업무 수정 (상태 변경 포함)
  DELETE /api/tasks/:id          업무 삭제
  POST   /api/tasks/:id/memos    업무에 메모 추가

회의록 (Meetings)
  GET    /api/meetings           전체 회의록
  GET    /api/meetings/:id       회의록 상세 (체크사항 포함)
  POST   /api/meetings           회의록 등록
  PUT    /api/meetings/:id       회의록 수정
  DELETE /api/meetings/:id       회의록 삭제
  PUT    /api/meetings/:id/checks/:checkId   체크사항 토글

공지 (Notices)
  GET    /api/notices            전체 공지 (읽음 상태 포함)
  POST   /api/notices            공지 등록
  DELETE /api/notices/:id        공지 삭제
  POST   /api/notices/:id/read   읽음 처리

메모 (Memos)
  GET    /api/memos?owner=지원   내 메모 목록
  POST   /api/memos              메모 등록
  DELETE /api/memos/:id          메모 삭제

캘린더 (Calendar)
  GET    /api/calendar?month=2026-03   월별 일정
  POST   /api/calendar           일정 등록

검색
  GET    /api/search?q=제안서     통합 검색 (업무+회의록+공지+메모)
```

### 프론트엔드 변경사항

현재 `localStorage` 코드를 `fetch` API 호출로 교체:

```javascript
// 변경 전 (localStorage)
const tasks = JSON.parse(localStorage.getItem('tasks'));

// 변경 후 (서버 API)
const response = await fetch('/api/tasks');
const tasks = await response.json();
```

// 변경 전
localStorage.setItem('tasks', JSON.stringify(tasks));

// 변경 후

## 배포 방법 (가장 쉬운 순서)

### 방법 1: 사내 PC에서 실행
```bash
# 1. 설치 (최초 1회)
npm install

# 2. 서버 실행
npm start

# 3. 접속
# 같은 네트워크: http://192.168.x.x:3000
```
→ 같은 와이파이/네트워크에 있으면 바로 접속 가능

### 방법 2: Railway 배포 (무료, 24시간)
```bash
# 1. Railway 가입 (GitHub 계정으로)
# 2. 프로젝트 연결
# 3. 자동 배포 → URL 받음
# → https://team-app-xxxx.railway.app
```

### 방법 3: Cloudflare Tunnel (무료, 자기 PC가 서버)
```bash
# 자기 PC에서 서버 돌리면서 외부 접속 허용
cloudflared tunnel --url http://localhost:3000
# → https://xxx.trycloudflare.com 주소 생성
```
